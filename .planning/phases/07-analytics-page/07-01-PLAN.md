    2. Create `apps/web/lib/compute-analytics.ts` with these exports:

    **Type: TimeRange**
    ```typescript
    export type TimeRange = "24h" | "7d" | "30d";
    ```

    **Type: TrendDataPoint**
    ```typescript
    export interface TrendDataPoint {
      readonly time: string;
      readonly count: number;
    }
    ```

    **Function: groupEventsByTime(events, range)**
    - Accepts `readonly SortEvent[]` and `TimeRange`
    - Returns `readonly TrendDataPoint[]`
    - For "24h": creates 24 hourly buckets going back 24 hours from now. Label format: hour in "h a" format (e.g., "12 PM", "1 AM"). Group events by their hour.
    - For "7d": creates 7 daily buckets going back 7 days. Label format: short weekday name (e.g., "Mon", "Tue"). Group events by calendar day.
    - For "30d": creates 30 daily buckets going back 30 days. Label format: short date (e.g., "Mar 1", "Mar 15"). Group events by calendar day.
    - Empty buckets get count: 0 (so the chart shows gaps, not compressed data).
    - Pure function, no side effects.

    **Function: getConfidenceColor(confidence)**
    - Accepts `number` (0 to 1 float)
    - Returns CSS class string:
      - confidence >= 0.8: returns "text-green-600"
      - confidence >= 0.5 and < 0.8: returns "text-amber-500"
      - confidence < 0.5: returns "text-red-500"

    **Function: computeTrendSummary(dataPoints)**
    - Accepts `readonly TrendDataPoint[]`
    - Returns `{ peak: number; average: number }` where:
      - peak = Math.max of all count values (0 if empty)
      - average = Math.round of mean of all count values (0 if empty)

    Follow the immutable patterns from compute-metrics.ts. All functions are pure with no mutations.
  </action>
