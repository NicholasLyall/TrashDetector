## Layout Contract

**Primary focal point:** CategoryBreakdownCard is the primary visual anchor on page load. It occupies the first grid cell in the 2-column chart row and draws the eye with its colored horizontal bars, making category distribution the first insight users absorb.

### Page Structure

```
AnalyticsPage (app/analytics/page.tsx):
  space-y-6
    [Page Header]                        -- h1 "Analytics" (text-xl font-bold) + subtitle
    [grid grid-cols-1 lg:grid-cols-2 gap-6]
      [CategoryBreakdownCard]            -- col-span-1, horizontal bar chart
      [SortingTrendCard]                 -- col-span-1, area chart with time selector
    [EventHistoryCard]                   -- full width, filterable event table
```

### CategoryBreakdownCard Internal Layout

```
Card
  CardHeader
    flex items-center justify-between
      CardTitle "Category Breakdown"
      [Total items count badge]          -- muted text, "241 total items"
  CardContent
    [Horizontal bar chart]               -- Recharts BarChart, horizontal layout
      4 bars (one per category)
      Each bar: category label left, count + percentage right
    [Legend row]                          -- flex gap-4, color dot + label pairs
```

- Chart height: 200px via `<ResponsiveContainer width="100%" height={200}>`
- Bar orientation: horizontal (category names on Y axis, counts on X axis)
- Bar radius: 4px rounded ends
- Bar height: 24px per bar
- Category labels: 14px regular, left-aligned on Y axis
- Count labels: 14px bold (`text-sm font-bold`), right of each bar

### SortingTrendCard Internal Layout

```
Card
  CardHeader
    flex items-center justify-between
      CardTitle "Sorting Trends"
      [Time range selector]              -- shadcn Select: "Last 24h" | "Last 7d" | "Last 30d"
  CardContent
    [Area chart]                         -- Recharts AreaChart, teal line + fill
      X axis: time labels (hours or dates)
      Y axis: event count
    [Summary row below chart]            -- flex justify-between, "Peak: X items" "Avg: Y items/hr"
```

- Chart height: 220px via `<ResponsiveContainer width="100%" height={220}>`
- Area fill: `hsl(174 62% 47% / 0.1)` (eco-teal at 10% opacity)
- Line stroke: `hsl(174 62% 47%)` (eco-teal), stroke-width 2px
- Dot on hover: 6px diameter circle, eco-teal fill, white 2px stroke
- X-axis format: "12pm", "1pm" for 24h view; "Mon", "Tue" for 7d view; "Mar 1", "Mar 2" for 30d view
- Grid: horizontal only, `var(--border)` color, dashed stroke
- Default selected range: "Last 24h"

### EventHistoryCard Internal Layout

```
Card
  CardHeader
    flex items-center justify-between
      CardTitle "Event History"
      [Showing count]                    -- "Showing 50 events" in label size
  CardContent
    space-y-4
      [CategoryFilter]                   -- horizontal row of filter pills
```

- Confidence column: percentage text, color-coded (green >= 80%, amber 50-79%, red < 50%)
- Fallback column: "Yes" in destructive color or "-" in muted
- Timestamp column: relative time ("2s ago", "5m ago") via existing `formatRelativeTime`
- Item column: display label from ITEM_NAMES or raw label fallback

## Interaction Contract

### Filter Behavior

- CategoryFilter pills toggle the event history table rows client-side
- Filtering is instant (no API call) -- filters the already-fetched events array
- Active filter pill gets category-specific colored border and background
- Switching filters does not trigger loading states
- "All" resets to show all events

### Time Range Selection

- Select dropdown in SortingTrendCard header controls the X-axis range
- Options: "Last 24h" (default), "Last 7d", "Last 30d"
- Changing selection re-renders the chart with the same data but different time grouping
- Data grouping: 24h groups by hour, 7d groups by day, 30d groups by day
- Since the API returns raw events, grouping is computed client-side

### Chart Interactions

- Recharts Tooltip on hover for both charts
- Bar chart tooltip: shows category name, count, and percentage
- Area chart tooltip: shows time label and event count
- No click-to-drill-down behavior
- No zoom/pan on charts

### Table Scroll

- Event history table body scrolls independently via `max-h-[480px] overflow-y-auto` on a wrapper div
- Table header stays fixed (sticky) during scroll: `sticky top-0 bg-card z-10`
- No pagination -- all fetched events shown with scroll
- Sort by timestamp descending (newest first, matching API default)

### Loading States

- Each card shows Skeleton placeholders independently while its data loads
- Chart skeletons: `<Skeleton className="h-[200px] w-full rounded-lg" />` matching chart height
- Table skeleton: 5 rows of `<Skeleton className="h-10 w-full" />` with gap-2
- Skeleton-to-empty-state delay: 5s (matching existing BackendEmptyState pattern from Phase 6 D-04)

### No Destructive Actions

This phase is read-only. No delete, reset, or modification actions exist.

Source: REQUIREMENTS.md ANLY-03 (filtering), existing interaction patterns from Phase 3/6.

## Data Contract

### API Endpoints Used

| Endpoint | Hook | Data | Refresh |
|----------|------|------|---------|
| GET /events?limit=50 | `useEvents(50)` | Event list for history table | 2s polling via SWR |
| GET /metrics/breakdown | `useBreakdown()` | Category counts for bar chart | 2s polling via SWR |
| GET /metrics | `useMetrics()` | Total items for context display | 2s polling via SWR |

### Client-Side Computed Data

| Computation | Input | Output |
|-------------|-------|--------|
| Time-grouped trend data | events array | `{ time: string; count: number }[]` grouped by hour/day |
| Filtered events | events array + active filter | Subset of events matching selected category |
| Confidence color | event.confidence | "text-green-600" (>=0.8), "text-amber-500" (0.5-0.79), "text-red-500" (<0.5) |

### Trend Data Computation

Events are grouped by timestamp into time buckets:
- **Last 24h**: group by hour, produce 24 data points, label format "12pm"
- **Last 7d**: group by calendar day, produce 7 data points, label format "Mon"
- **Last 30d**: group by calendar day, produce 30 data points, label format "Mar 1"

This computation lives in a pure function `groupEventsByTime(events, range)` in `lib/compute-analytics.ts`.

Source: existing `useEvents`, `useBreakdown`, `useMetrics` hooks, `lib/compute-metrics.ts` pattern.

## States

### Default State (data loaded)

All three cards render with live data from the backend. Charts display data points, table shows event rows.

### Loading State

Each card independently shows Skeleton shimmer while its hook is in `isLoading` state. Cards load independently -- the bar chart may render before the trend chart.

### Empty State (no events)

When API returns zero events:
- CategoryBreakdownCard: centered text, no chart rendered
- SortingTrendCard: centered text, flat zero line
- EventHistoryCard: centered text, no table rows

### Error State

When API is unreachable:
- After 5s delay, show BackendEmptyState component (reuse existing pattern)
- Individual cards can show error independently

Source: existing loading/empty/error patterns from `waste-composition-card.tsx` and Phase 6.

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Chart accessibility | Each chart card has `aria-label` describing the data (e.g., "Bar chart showing waste category distribution") |
| Filter pills | Use `role="radiogroup"` on container, `role="radio"` + `aria-checked` on each pill |
| Table semantics | Use semantic `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` elements (shadcn Table provides these) |
| Color alone | Confidence values show both color and text percentage -- color is supplementary, not sole indicator |
| Keyboard navigation | Filter pills focusable and activatable via Enter/Space, tab order follows visual order |
| Reduced motion | Chart animations disabled when `prefers-reduced-motion: reduce` |
| Screen reader | Chart data also accessible via table (event history serves as data table alternative to visual charts) |

Source: Phase 2/3 accessibility patterns, WCAG 2.1 AA requirements.
