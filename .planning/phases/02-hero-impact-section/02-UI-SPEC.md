## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px | 400 (regular) | 1.5 | Stat card labels ("Recycled", "CO2 Saved", "Total Items Sorted"), unit suffixes |
| Heading | 20px | 700 (bold) | 1.3 | Stat card metric values ("68%", "21 kg", "241") |
| Display | 28px | 700 (bold) | 1.2 | Motivational headline ("Great job! You've diverted **12.4 kg** of waste today.") |

Font: `var(--font-sans)` (Geist) for all text. The motivational message uses Display size. The dynamic number within the motivational message ("12.4 kg") is rendered in eco-green color to visually pop. Both Heading and Display share the same weight (700 bold) to keep the weight palette minimal while maintaining clear hierarchy through size difference (20px vs 28px).

### Eco Illustration Palette

| Token | Value | Usage |
|-------|-------|-------|
| `--eco-green` | `hsl(142 71% 45%)` | Leaf SVGs, metric number highlight, stat card icons |
| `--eco-teal` | `hsl(174 62% 47%)` | Cloud accent tints, secondary leaf colors, gradient midpoint |
| `--eco-blue` | `hsl(199 89% 48%)` | Sky gradient top, globe-free background illustration |

### Hero Card Gradient

The hero card background uses a top-to-bottom gradient:
- Top: `hsl(199 89% 90%)` (light sky blue, derived from --eco-blue at 90% lightness)
- Middle: `hsl(174 62% 88%)` (light teal, derived from --eco-teal at 88% lightness)
- Bottom: `hsl(142 71% 90%)` (light green, derived from --eco-green at 90% lightness)

Opacity: 100% on the gradient. SVG illustrations sit on top with full opacity.

### Accent Reserved For

- Motivational message dynamic number ("12.4 kg") — eco-green text
- Stat card icons — eco-green, eco-teal, eco-blue (one per card)
- Floating leaf SVG fills — eco-green at varying opacities (0.3 to 0.7)
- Never: backgrounds of stat cards (those stay white), body text, borders

## Component Inventory

### 1. HeroImpactSection (container)

- **Type:** Feature component at `components/dashboard/hero-impact-section.tsx`
- **Element:** Single `<Card>` with custom gradient background and overflow-hidden
- **Dimensions:** Full width of content area (inherits from parent `space-y-6` in app-shell)
- **Min height:** 280px to ensure illustration has room
- **Border radius:** `rounded-xl` (matches shadcn Card default, --radius based)
- **Shadow:** `shadow-sm` (subtle, not heavy)
- **Ring:** `ring-1 ring-foreground/5` (softer than default Card ring)
- **Overflow:** `overflow-hidden` (clip illustration SVGs that extend to edges)
- **Internal layout:** Flexbox column, items-center, with padding `p-6 pt-12`

### 2. HeroIllustration (background layer)

- **Type:** Presentational component at `components/dashboard/hero-illustration.tsx`
- **Position:** `absolute inset-0` inside the hero card (card must be `relative`)
- **Contains:** Inline SVG elements for sky, clouds, and floating leaves
- **z-index:** `z-0` (behind text content)
- **Clouds:** 2-3 white cloud shapes at varying positions, opacity 0.6-0.8
- **Leaves:** 4-6 leaf SVG shapes scattered at card edges, eco-green fills at 0.3-0.7 opacity
- **Animation:** Leaves use CSS `@keyframes float` — gentle vertical bob (translateY +/- 6px) over 3-5 seconds, ease-in-out, infinite. Each leaf has a staggered `animation-delay` (0s, 0.5s, 1s, 1.5s, etc.)
- **No globe:** Per D-07, exclude globe from illustration

### 3. MotivationalMessage (headline)

- **Type:** Inline within HeroImpactSection
- **Element:** `<p>` tag
- **Position:** Centered horizontally, `z-10` (above illustration)
- **Text:** "Great job! You've diverted **{value} kg** of waste today."
- **"Great job!"** — Display size (28px), bold (700), foreground color
- **"{value} kg"** — Display size (28px), bold (700), eco-green color (`text-[hsl(var(--eco-green))]`)
- **Rest of text** — Display size (28px), bold (700), foreground color
- **Alignment:** `text-center`
- **Max width:** `max-w-lg` to prevent overly wide single-line on large screens

### 4. StatCard (x3)

- **Type:** Sub-component within HeroImpactSection (can be inline, not separate file)
- **Element:** `<Card>` from shadcn/ui
- **Layout:** 3 cards in a horizontal row, equal width, `grid grid-cols-3 gap-4` on desktop
- **Responsive:** On mobile (`< md`), stack vertically `grid-cols-1` or use `flex flex-col gap-3`
- **Position:** Below motivational message, `z-10`, with `mt-8` spacing
- **Background:** White (`bg-card` / `bg-white`)
- **Shadow:** `shadow-sm`
- **Border:** default Card ring (`ring-1 ring-foreground/10`)
- **Internal layout:** Flex row, items-center, `p-4 gap-3`
- **Icon:** 24x24px Lucide icon in a 40x40px rounded-lg container with light eco-color background
- **Content:** Flex column — metric value (Heading 20px/700 bold) on top, label (Body 14px/400, muted-foreground) below

- **Values:** Hardcoded object with today's mock data

## Responsive Behavior

| Breakpoint | Layout Change |
|------------|---------------|
| `>= md` (768px+) | 3 stat cards in horizontal row (`grid-cols-3`), Display text at 28px |
| `< md` (mobile) | Stat cards stack vertically (`grid-cols-1`), Display text reduces to 22px, hero card padding reduces to `p-4 pt-8` |
| `< sm` (375px) | Same as mobile, motivational message drops to 20px for fit |

Illustration scales with the card -- clouds and leaves are positioned with percentage-based SVG coordinates, not fixed px. On mobile, some leaves may be hidden (`hidden sm:block`) to reduce visual clutter.

## States

### Default State (mock data present)

The hero section renders with hardcoded mock values. This is the only state for Phase 2 since backend integration comes in Phase 6.

### Loading State (Phase 6 prep)

Not required in Phase 2. When data integration arrives, a skeleton shimmer can replace metric values. Design the stat card content area to accept either a number or a skeleton placeholder without layout shift.

### Empty State

Not applicable for Phase 2 -- mock data is always present. For future (Phase 6): if the backend returns zero events, the motivational message changes to the empty state copy below.

### Error State

Not applicable for Phase 2 -- no data fetching. Reserved for Phase 6.

## Interaction Contracts

### Animations

| Element | Animation | Duration | Easing | Trigger |
|---------|-----------|----------|--------|---------|
| Floating leaves | `translateY(-6px)` to `translateY(6px)` | 3-5s per leaf | ease-in-out | On mount, infinite loop |
| Cloud drift (optional) | `translateX(-4px)` to `translateX(4px)` | 8-12s | ease-in-out | On mount, infinite loop |

Animation is CSS-only (no JS animation libraries). Each leaf has a unique duration and delay to avoid synchronized bobbing:
- Leaf 1: duration 3s, delay 0s
- Leaf 2: duration 4s, delay 0.7s
- Leaf 3: duration 3.5s, delay 1.2s
- Leaf 4: duration 5s, delay 0.3s
- Leaf 5: duration 4.2s, delay 1.8s

### Accessibility

- Stat card icons are decorative: `aria-hidden="true"` on each icon
- SVG illustration is decorative: `aria-hidden="true"` on the illustration container, `role="presentation"`
- Motivational message is a `<p>` not a heading (the page `<h1>` is elsewhere or this becomes the `<h1>` -- if it becomes `<h1>`, use `<h1>` element)
- Color contrast: all text must meet WCAG AA (4.5:1 for body, 3:1 for large text). The eco-green on light gradient must be checked -- if contrast fails, use darker foreground for metric highlight
- `prefers-reduced-motion`: wrap leaf/cloud animations in `@media (prefers-reduced-motion: no-preference)` -- disable all animation when user prefers reduced motion

### Dark Mode

The hero card should work in dark mode. Strategy:
- Gradient background shifts to darker values: top `hsl(199 89% 15%)`, mid `hsl(174 62% 12%)`, bottom `hsl(142 71% 15%)`
- Cloud SVGs shift to `opacity: 0.15` (subtle glow)
- Leaf SVGs shift to `opacity: 0.4` with slightly brighter fills
- Stat cards use `bg-card` (which resolves to dark card color in dark mode via shadcn tokens)
- Text colors use `foreground` token (auto-switches)
- The eco-green highlight in the motivational message stays green but adjusts: use `hsl(142 71% 55%)` in dark mode for readability
