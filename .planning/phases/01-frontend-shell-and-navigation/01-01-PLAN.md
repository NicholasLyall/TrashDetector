  <action>
    Initialize shadcn/ui in the apps/web/ directory:

    ```bash
    cd apps/web && npx shadcn@latest init --defaults
    ```

    This creates:
    - components.json with path aliases pointing to @/components and @/lib
    - lib/utils.ts with the cn() utility function (uses clsx + tailwind-merge)

    After init, verify components.json has these settings:
    - style: "new-york" (or "default" -- use whichever shadcn defaults to)
    - aliases.components: "@/components"
    - aliases.utils: "@/lib/utils"

    Then install the core shadcn/ui components that will be needed across the app shell and subsequent phases:

    ```bash
    cd apps/web && npx shadcn@latest add button card badge tooltip separator avatar sheet
    ```

    These components provide:
    - button: Navigation items, actions
    - card: Dashboard section containers (CardHeader, CardTitle, CardDescription, CardContent)
    - badge: Status badges, bin badges (Recycled, Composted, Trash)
    - tooltip: Icon hover labels in sidebar
    - separator: Visual dividers in sidebar and content
    - avatar: Admin user avatar in sidebar and top bar
    - sheet: Mobile sidebar drawer overlay

    Verify lib/utils.ts exports the cn function:
    ```typescript
    import { clsx, type ClassValue } from "clsx"
    import { twMerge } from "tailwind-merge"

    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs))
    }
    ```
