  <action>
    Create `apps/web/lib/compute-metrics.ts` with three pure exported functions:

    1. `computeKpiMetrics(events: readonly SortEvent[]): KpiMetrics`
       - Returns object: `{ totalItems: number, recyclingRate: number, avgConfidence: number, fallbackRate: number }`
       - `totalItems` = events.length
       - `recyclingRate` = Math.round((non-trash count / total) * 100), where non-trash means label !== "trash"
       - `avgConfidence` = Math.round((sum of all confidence values / total) * 100)
       - `fallbackRate` = Math.round((fallback_used true count / total) * 100)
       - Return all zeros if events is empty (avoid division by zero)

    2. `computeCategoryBreakdown(events: readonly SortEvent[]): CategoryBreakdownEntry[]`
       - Returns array of `{ category: WasteCategory, count: number, percentage: number, color: string }`
       - Group events by `label` field
       - For each category in CATEGORY_CONFIG keys, compute count and percentage (rounded)
       - `color` = `CATEGORY_CONFIG[category].badgeColor` value (the HSL string)
       - Sort descending by count
       - Only include categories with count > 0

    3. `computeModelStats(events: readonly SortEvent[]): ModelStats`
       - Returns object: `{ avgConfidence: number, uncertainRate: number, fallbackRate: number }`
       - `avgConfidence` = Math.round((sum of confidence / total) * 100)
       - `uncertainRate` = Math.round((events with confidence < 0.5 count / total) * 100)
       - `fallbackRate` = Math.round((events with fallback_used true / total) * 100)
       - Return all zeros if events is empty

    Export the type interfaces: `KpiMetrics`, `CategoryBreakdownEntry`, `ModelStats`.

    Create `apps/web/lib/compute-metrics.test.ts` with tests for all behaviors listed above.
    Import `sortEventsMockData` from mock-data for integration-style tests, plus unit tests with hand-crafted arrays.
