## Color

### Surface Colors (60/30/10)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` oklch(1 0 0) / oklch(0.145 0 0) dark | Page background behind feed section |
| Secondary (30%) | `var(--card)` oklch(1 0 0) / oklch(0.205 0 0) dark | LiveFeedSection card, LatestItemCard inner card |
| Accent (10%) | `hsl(var(--eco-green))` = hsl(142 71% 45%) | Green status dot, tree badge icon, newest-item highlight ring |

Accent reserved for: green status dot pulse, TreePine icon color, newest event left-border highlight. Never used for badges or general decoration.

### Category Badge Colors (Phase 3 Specific)

These are semantic category colors, not the accent color. Each has a foreground (text + icon) and a background (12% opacity fill).

| Category | Foreground | Background | Dark Mode Foreground | Dark Mode Background |
|----------|-----------|------------|---------------------|---------------------|
| Paper & Cardboard (Blue) | `hsl(210 90% 40%)` | `hsl(210 90% 40% / 0.12)` | `hsl(210 90% 60%)` | `hsl(210 90% 60% / 0.15)` |
| Metal & Glass (Amber) | `hsl(38 92% 45%)` | `hsl(38 92% 45% / 0.12)` | `hsl(38 92% 60%)` | `hsl(38 92% 60% / 0.15)` |
| Plastic (Green) | `hsl(142 71% 35%)` | `hsl(142 71% 35% / 0.12)` | `hsl(142 71% 55%)` | `hsl(142 71% 55% / 0.15)` |
| Landfill (Red) | `hsl(0 72% 45%)` | `hsl(0 72% 45% / 0.12)` | `hsl(0 72% 60%)` | `hsl(0 72% 60% / 0.15)` |

Dark mode values use +15-20 lightness to maintain WCAG AA contrast (4.5:1) against dark card backgrounds.

Source: CONTEXT.md D-02 (blue/amber/green/red families), RESEARCH.md Pitfall 2 (dark mode badge contrast).

### Semantic Colors

| Role | Value | Usage |

| Destructive | `var(--destructive)` oklch(0.577 0.245 27.325) | Not used this phase (no destructive actions) |
| Muted foreground | `var(--muted-foreground)` oklch(0.556 0 0) | Timestamps, secondary text |

## Interaction Contract

### Status Dot Pulse

- Element: 10px circle, `bg-green-500`
- Animation: CSS `animate-ping` on an overlapping circle at 75% opacity, solid circle underneath
- Reduced motion: `motion-reduce:hidden` on the ping layer
- Purpose: Communicates "system is alive" -- static in this phase (no actual polling until Phase 6)

### Scroll Behavior

- RecentItemsList: native browser scroll via `overflow-y-auto max-h-[320px]`
- No custom scrollbar styling
- No infinite scroll or pagination -- all 7-9 items loaded at once

### No Interactive Elements

This phase is purely presentational. No click handlers, no hover states beyond default cursor, no form inputs, no modals. The components render static mock data.

Source: RESEARCH.md key insight -- "Every interaction is CSS-only."

## Mock Data Contract

### SortEvent Interface

Must match backend GET /events response shape exactly for clean Phase 6 swap:

```typescript
interface SortEvent {
  readonly id: string;              // "evt-001"
  readonly timestamp: string;       // ISO 8601, relative to Date.now()
  readonly label: WasteCategory;    // "plastic" | "paper_cardboard" | "metal_glass" | "trash"
  readonly routed_bin: WasteCategory;
  readonly confidence: number;      // 0-1 float (display as percentage)
  readonly image_url: string | null; // null for mock, URL for live
  readonly waste_diverted_kg: number;
  readonly co2_saved_kg: number;
  readonly fallback_used: boolean;
  readonly source_device_id: string; // "pi-001"
}
```

### Mock Data Values

- 8-10 events with varied categories, confidence (0.41 to 0.97 range), and timestamps
- Timestamps generated relative to `Date.now()` at module load: 2s, 5s, 14s, 31s, 59s, 2m, 5m, 12m, 30m ago
- CO2 values inflated to produce meaningful tree count: total CO2 across events should sum to ~50-70 kg so tree badge shows 2-3 trees
- First event (newest) becomes the LatestItemCard; remaining populate RecentItemsList
- Include at least one fallback item (confidence < 0.7, `fallback_used: true`, `routed_bin: "trash"`)

Source: CONTEXT.md D-09, RESEARCH.md Pitfall 3 (stale timestamps), Open Question 3 (inflated CO2).

## SVG Illustration Contract

### General Rules

- ViewBox: `0 0 120 120` for all category illustrations
- Style: flat, eco-themed, matching `hero-illustration.tsx` palette (sky blues, leaf greens, warm earth tones)
- Max complexity: 30 lines of SVG markup per illustration
- Use `fill` not `stroke` for primary shapes (scales well at both 200px and 40px per D-07)
- All illustrations wrapped in a shared `CategoryIllustration` component that accepts `category` and `size` props
- "Simulated" badge: absolute positioned, `bg-gray-500/70 text-white text-[10px] font-medium px-1.5 py-0.5 rounded`

### Per-Category Illustrations

| Category | Subject | Primary Colors |
|----------|---------|----------------|
| paper_cardboard | Stacked papers/cardboard box | Light tan `hsl(35 50% 75%)`, brown `hsl(30 40% 50%)` |
| metal_glass | Glass bottle or soda can | Light teal `hsl(174 50% 70%)`, silver `hsl(0 0% 75%)` |
| plastic | Plastic bottle | Light blue `hsl(199 60% 70%)`, blue `hsl(199 89% 48%)` |
| trash | Trash bag | Gray `hsl(0 0% 60%)`, dark gray `hsl(0 0% 40%)` |

Source: CONTEXT.md D-05 through D-08, RESEARCH.md Pitfall 4 (SVG sizing).

## Accessibility Contract

| Concern | Implementation |
|---------|---------------|
| Badge contrast | All badge foreground/background pairs must meet WCAG AA 4.5:1 contrast ratio |
