**apps/api/seed.py** -- Idempotent wipe-and-reseed script per D-16 through D-19:

Create a script that can be run as `python -m apps.api.seed` (add `if __name__ == "__main__"` block).

**Structure:**

1. `SEED_DEVICE_ID = "d0000000-0000-0000-0000-000000000001"` -- fixed UUID for the default device so upsert is idempotent.

2. `SEED_DEVICE = {"id": SEED_DEVICE_ID, "name": "Pi Sorter Alpha", "location_name": "Lab Station 1", "status": "active"}` -- per D-19

3. `def generate_seed_events(count: int = 40) -> list[dict]`:
   - Generate `count` events spread across the last 3 hours per D-17
   - Distribution across categories: roughly 30% plastic, 25% paper_cardboard, 20% metal_glass, 25% trash (weighted random, not exact -- use `random.choices` with weights `[30, 25, 20, 25]`)
   - For each event:
     - `label = random.choices(VALID_CATEGORIES, weights=[25, 20, 30, 25], k=1)[0]` (paper_cardboard, metal_glass, plastic, trash)
     - `confidence = round(random.uniform(0.45, 0.99), 2)` -- varied, some below 0.7 threshold
     - Apply `apply_fallback(label, confidence)` to get `(final_label, fallback_used)` -- reuse from event_service
     - Apply `compute_impact(final_label)` to get `(waste_diverted_kg, co2_saved_kg)` -- reuse from event_service
     - `timestamp = now - timedelta(minutes=random.randint(1, 180))` -- spread across 3 hours per D-17
     - `id = str(uuid.uuid4())`
     - `image_url = None` -- per D-18: placeholder images or skip images. Use None for seed data.
     - `source_device_id = SEED_DEVICE_ID`
   - Return list of event dicts sorted by timestamp descending (newest first)
   - Use `random.seed(42)` for reproducible output during testing (but the function should accept an optional seed parameter)

4. `def wipe_data()`:
   - Delete all rows from sort_events: `supabase.table("sort_events").delete().neq("id", "00000000-0000-0000-0000-000000000000").execute()`
   - Note: Supabase requires a filter for delete -- using `.neq("id", "00000000-...")` deletes all rows since no row has that UUID
   - Do NOT delete devices -- upsert handles that

5. `def seed(count: int = 40)`:
   - Print "Wiping existing events..."
   - Call `wipe_data()`
   - Print "Upserting default device..."
   - `supabase.table("devices").upsert(SEED_DEVICE).execute()`
   - Print f"Generating {count} seed events..."
   - `events = generate_seed_events(count)`
   - Print "Inserting seed events..."
   - Insert events in a single batch: `supabase.table("sort_events").insert(events).execute()`
   - Print f"Seed complete: {count} events inserted"

6. `if __name__ == "__main__":` block:
   - Import and call `seed()`
   - Wrap in try/except to print helpful error if Supabase credentials are missing

Import `apply_fallback` and `compute_impact` from `apps.api.services.event_service` to ensure the seed data uses the exact same business logic as real events.
